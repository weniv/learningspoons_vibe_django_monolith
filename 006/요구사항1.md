나는 Django를 이제 막 배우는 초보 개발자야. 강사님이 `명령어_백업.md` 파일을 이전시간에 작성하여 주었어.

이 파일을 참고해서 실제 동작하는 Django 웹 애플리케이션 지시사항 파일인 `명령어.md` 파일을 만들고 싶어. 이 파일에 개발되어야 하는 사항과 API는 아래와 같아.

# API 설계
* Main App
- 메인 페이지: `/`
- 소개 페이지: `/about/`
- 연락처 페이지: `/contact/`

* Accounts App
- 회원가입: `/accounts/signup/`
- 로그인: `/accounts/login/`
- 로그아웃: `/accounts/logout/`

* Blog App
- 블로그 글 목록: `/blog/`
- 블로그 글 상세: `/blog/<int:pk>/`
- 블로그 글 생성: `/blog/create/`
- 블로그 글 수정: `/blog/<int:pk>/update/`
- 블로그 글 삭제: `/blog/<int:pk>/delete/`
- 댓글 생성: `/blog/<int:post_pk>/comment/create/`
- 댓글 수정: `/blog/comment/<int:pk>/update/`
- 댓글 삭제: `/blog/comment/<int:pk>/delete/`
- tag 검색: `/blog/tag/<str:slug>/`
- title 검색: `/blog/?search=<str:query>`

# 모델 설계
* 좋아요, 조회수, 이미지 등이 포함된 블로그 모델
* 댓글 모델
* 사용자 별명, 사용자 이미지 등이 포함된 사용자 모델 (Django 기본 User 모델 확장)

이러한 방식을 직접만들어볼 수 있도록 아래 파일을 순서대로 작성해줘. 파일을 만들 때에는 아래 사항을 고려해줘.

1. 명령어를 설계할 때에는 CBV 방식으로 작성해줘. 
2. tailwindcss를 사용하여 기본적인 스타일링도 포함해줘.
3. templates은 각 앱 안에 있는 것이 아니라 프로젝트의 최상위 templates 폴더에 위치하도록 해줘.
4. 블로그 글 목록에서는 전체 태그를 볼 수 있게 해줘. 클릭하면 관련된 태그가 포함된 글들만 필터링되도록 해줘.

우선은 1번부터 작성해줘. 순차적으로 내가 지시할게. 

1. `API_설계.md`: 각 기능에 대한 API 설계 문서
2. `모델_설계.md`: 데이터베이스 모델 설계 문서, ERD 머메이드로 표현
3. `명령어.md`: 실제 Django 명령어 및 코드 작성 지시사항
